# apix Core Vault

This directory models the `apix-sh/vault` repository layout.

## Repository Purpose

The core vault stores curated API docs generated by `apix import` and indexed by `registry.json`.

## Required Layout

```text
./
├── registry.json
├── <namespace>/
│   └── <version>/
│       ├── _metadata.md
│       ├── _types/
│       └── <route path>/<METHOD>.md
└── .metadata/
    ├── catalog.yaml
    ├── policies/
    ├── scripts/
    └── sources/
```

Root docs allowed:

- `README.md`
- `CONTRIBUTION.md`

See CLI-shipped spec: `../cli/docs/vault-repo-spec.md`.

## Maintainer Workflow (Bootstrap and Ongoing)

1. Sync local checkout to latest and create a branch:
   - `git checkout main && git pull`
   - `git checkout -b feat/<namespace>-<version>`
2. Import directly into this worktree:
   - `apix import <openapi_spec> --name <namespace> --output /path/to/vault`
   - if replacing an existing version: add `--overwrite`
3. Rebuild index:
   - `apix registry rebuild --path /path/to/vault`
4. Validate diffs (`registry.json` + content tree).
5. Open PR with summary (new namespace/version, changed route/type counts, source spec URL).

## Notes

- Keep namespace/version updates focused and deterministic.
- Prefer one namespace/version update per PR.
- Use `--overwrite` only when intentionally replacing an existing generated version tree.
