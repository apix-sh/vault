# secret-scanning-pattern-override

## Properties

| Property | Required | Type | Description |
| :------- | :------: | :--- | :---------- |
| `token_type` | No | string | The ID of the pattern. |
| `custom_pattern_version` | No | string | The version of this pattern if it's a custom pattern. |
| `slug` | No | string | The slug of the pattern. |
| `display_name` | No | string | The user-friendly name for the pattern. |
| `alert_total` | No | integer | The total number of alerts generated by this pattern. |
| `alert_total_percentage` | No | integer | The percentage of all alerts that this pattern represents, rounded to the nearest integer. |
| `false_positives` | No | integer | The number of false positive alerts generated by this pattern. |
| `false_positive_rate` | No | integer | The percentage of alerts from this pattern that are false positives, rounded to the nearest integer. |
| `bypass_rate` | No | integer | The percentage of blocks for this pattern that were bypassed, rounded to the nearest integer. |
| `default_setting` | No | string | The default push protection setting for this pattern. Allowed values: disabled, enabled |
| `enterprise_setting` | No | string | The push protection setting for this pattern set at the enterprise level. Only present for partner patterns when the organization has a parent enterprise. Allowed values: not-set, disabled, enabled |
| `setting` | No | string | The current push protection setting for this pattern. If this is `not-set`, then it inherits either the enterprise setting if it exists or the default setting. Allowed values: not-set, disabled, enabled |
